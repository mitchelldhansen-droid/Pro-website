The Day My Game Booted in My Own Hands

Jul 21, 2026 · 3 min read

The honest truth is that when I went to run the game I'm working on onto my Steam Deck today, I fully expected it to fail. Not because I doubted the code — because I couldn't picture it any other way.

For months, it has only ever existed in one place: a debug window in the corner of my PC, wrapped inside the Godot editor. Tiny. Contained. One-dimensional. When you see something in exactly one context for long enough, you stop imagining it anywhere else. So when I went to boot it on the Deck, the voice in my head said, flatly, “there's no way this boots.”

Then the title screen came up. And I about dropped my Deck.

Hand clasped over my face. Shouting for anyone nearby to come see. And then I did the thing I had never actually done with my own game before: I leaned back and just played it — on a couch, in my hands, on a device built for playing games. Because it was a game now, and I wanted to.

One command to the Deck

Getting there was its own project. This was the first time I'd ever exported the build at all, and I didn't want to fight the process by hand every time. So I wrote a script that does the whole thing in a single command: it sets the export config so we know where the build is going, pulls the most recent version from main on remote, exports that build, guards against a suspiciously small or large package so a broken export can't slip through, and then pushes it from my computer to the Deck over SSH. It took some work and a couple of broken builds to get right. But eventually — one command, and my game lands on my handheld.

Part of something larger

Here's what actually shifted. Up to today, it had lived in that one corner of my machine, always behind the Godot bootup wrapper. I'd never just launched my game and been dropped straight into playing it. Doing that — on the same Steam Deck where I play the cozy sims and roguelites I love — made something click that no amount of editor time ever did. People from all walks of life, with all kinds of backgrounds, on all kinds of devices, could pick this up someday. It stopped being a project just for me. It quietly became part of something larger. A community.

I'll be straight about the part I don't usually say out loud: for most of this, I've felt like I was faking it. Like “game developer” was an elaborate costume I'd stitched together and was hoping no one would look at too closely.

Today that fell away. I'm a game developer.

Standing next to giants

It was surreal in another way too. Sitting there, I thought: if I'm going to stand next to giants — the developers whose games are already in this library — I'd better make damn sure I'm ready to be compared to them. This isn't a project I'll submit for a grade one day, where a teacher can squint at where I was going and I can explain my intentions to the class afterward. This is a game I hope people will spend real money on and escape into for dozens, maybe hundreds of hours. So it had better be a good one.

Production has always been my foundation. Days like today are teaching me that engineering is becoming my edge.

Nine months ago, I couldn't export a build. Today, my game booted in my own hands — and I got to play it.

Back at it tomorrow.
